# 9. Contraintes de développement

- Respect des normes d'accessibilité (RGAA partiellement).
- Internationalisation : interface en français (architecture préparée pour ajout d'autres langues).
- Compatibilité navigateurs : dernières versions de Chrome, Firefox, Edge, Safari.
- Code source commenté et documenté (docstring, README, wiki technique).
- Tests unitaires et fonctionnels couvrant au moins 80 % du code critique.
- Gestion des migrations de base de données versionnées.
- Conformité aux maquettes UI/UX validées.
